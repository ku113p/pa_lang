**Change types**:
- **encoding-only** (find_zero, find_delim, fibonacci): Same instruction sequence; `ffz!`→`ffz fA` or `mv!`→`mv xL` compact cell replaces extended instruction.
- **new-idiom** (min_byte): `cm! r3,r1; jn qB; mv xF; @F0:` (7B, 4 instructions + label) replaced by `cm cE; cs xF` (4B, 2 instructions). Branchless conditional select.
- **(none)** (sum_bytes, xor_buffer, compare_bufs): Already fully compact in v0.1.

### Encoding vs Idiom Breakdown (min_byte)

|---------|----------|----------|-------|

| v0.1 algorithm + v0.2 encoding | 21B | 0 | cm cF replaces cm! (-1B encoding gain) |
| v0.2 algorithm + v0.2 encoding | 19B | 0 | cm cE + cs xF replaces branch-over (-2B idiom gain) |

## Impact of v0.2 Changes

### What Changed
- Added 2 compact predicate extraction cells (f-family: fA, fB)
- Added 6 compact ordered compare cells (c-family: cA-cF)
- Added 3 additional register pair cells (x-family: xL, xM, xN)
- Added 1 new opcode: `cs` (conditional select)
- Promoted `cm` and `ffz` from extended-only to compact+extended

### Cost of v0.2 Extensions

| Metric | v0.1 | v0.2 | Change |
|--------|------|------|--------|
| Opcode count | 14 | 15 | +1 |
| Compact cell count | 27 | 38 | +11 |
| Encoding space used | 27/256 (10.5%) | 38/256 (14.8%) | +4.3% |
| Extended instructions (7 original kernels) | 4 | 0 | -4 |
| Bytes/instruction | 1.92 | 1.85 | -0.07 |

Cell table growth is modest (38/256 = 14.8%). Well below the 25% threshold that would warrant concern about encoding complexity.

### Effect on Metrics

| Metric | v0.1 | v0.2 | Change |
|--------|------|------|--------|
| PA/x86 bytecode (weighted, 7 original) | 59% | 56% | -3pts |
| PA/Wasm bytecode (weighted) | 32% | 30% | -2pts |
| Bytes/instruction | 1.92 | 1.85 | -0.07 |
| Source ratio (PA/x86) | 42% | 39% | -3pts |

### Key Observations
- **Extended instructions eliminated** from all 7 original kernels (4→0). Every original kernel is now fully compact.
- **min_byte improved most** (-3B, 14% reduction) — the only kernel with a new-idiom change. The branchless `cm+cs` pattern is competitive with x86 `cmp+cmovb`.
- **External Class A kernels** also benefit: memchr, strlen, find_mismatch now use compact `ffz fA` (0 extended, was 1 each).
- **Class B/C kernels mostly unchanged** — their friction comes from missing predicate OR, set-membership, and reverse scan, which v0.2 deliberately does not address (below frequency threshold).
- **No regressions**: no kernel increased in bytecode size.
